# 模型存档功能使用说明

## 功能概述

模型存档功能允许教师创建和管理神经认知诊断模型的训练存档，包括：
- 创建模型存档
- 上传训练数据（知识点映射文件、训练集、验证集、测试集）
- 自动训练模型并保存最佳模型
- 在创建专栏时选择使用的模型存档

## 数据库修改

在首次使用前，需要执行以下 SQL 语句更新数据库：

```sql
-- 创建模型存档表
CREATE TABLE IF NOT EXISTS model_archives (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL UNIQUE,
    word_emb_path VARCHAR(200),
    diagnosis_model_path VARCHAR(200),
    knowledge_mapping_path VARCHAR(200),
    teacher_id INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    training_log TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (teacher_id) REFERENCES users(id)
);

-- 为 exam_columns 表添加 archive_id 字段
ALTER TABLE exam_columns ADD COLUMN archive_id INTEGER;

-- 为 exam_columns 表添加 question_knowledge 字段
ALTER TABLE exam_columns ADD COLUMN question_knowledge TEXT;
```

## 使用步骤

### 1. 创建初始存档（已完成）

已自动创建初始存档，路径：`backend/uploads/archives/initial_archive_YYYYMMDD_HHMMSS/`

初始存档包含：
- `knowledge_mapping.txt` - 知识点映射文件
- `initial_archive_YYYYMMDD_HHMMSSN` - 诊断模型（基于 model_epoch28）
- `initial_archive_YYYYMMDD_HHMMSSW` - 词嵌入模型
- 训练数据文件

### 2. 创建新存档

1. 登录教师账号
2. 点击顶部导航栏的"🗄️ 模型存档管理"
3. 点击"➕ 创建新存档"
4. 输入存档名称（如：2024 届模型）
5. 点击"创建"

### 3. 上传数据并训练

1. 在存档列表中找到刚创建的存档（状态为"待训练"）
2. 点击"📤 上传数据并训练"
3. 上传以下文件：
   - **知识点映射文件**（必需）：`knowledge_mapping.txt` 格式
   - **训练数据文件**（可选）：JSON 格式，支持以下命名：
     - `train_*.json` - 训练集
     - `val_*.json` 或 `valid_*.json` - 验证集
     - `test_*.json` - 测试集
4. 点击"开始训练"

训练过程：
- 系统会自动运行 `NeuralCDM_plus-main/full_train.py`
- 训练完成后会自动选择最佳 epoch 的模型
- 模型文件会保存到存档目录，命名为：`存档名 +W`（词嵌入）和 `存档名+N`（诊断模型）

### 4. 创建专栏时选择存档

1. 点击"📋 创建专栏"
2. 输入专栏名称
3. **在"选择模型存档"下拉框中选择一个已训练完成的存档**（必选项）
4. 上传试题图片
5. 点击"创建专栏"

### 5. 查看存档详情

1. 在存档列表中找到已完成的存档
2. 点击"👁️ 查看详情"
3. 可以查看：
   - 存档状态
   - 模型文件路径
   - 训练日志

## 数据格式要求

### knowledge_mapping.txt 格式

```
1：知识点名称 1
2：知识点名称 2
3：知识点名称 3
...
```

### 训练数据 JSON 格式

每个 JSON 文件应包含题目 - 知识点 - 得分的三元组：

```json
[
  {
    "student_id": 1,
    "exercise_id": 1,
    "knowledge_codes": [1, 2],
    "score": 1
  },
  {
    "student_id": 1,
    "exercise_id": 2,
    "knowledge_codes": [3],
    "score": 0
  }
]
```

## 存档状态说明

- **待训练**：存档已创建，等待上传数据
- **训练中**：正在训练模型
- **已完成**：模型训练完成，可以使用
- **失败**：训练失败，查看训练日志了解原因

## 注意事项

1. **存档名称唯一**：不能创建同名存档
2. **训练时间**：模型训练可能需要较长时间，请耐心等待
3. **删除限制**：有专栏正在使用的存档无法删除
4. **模型路径**：评语生成时会自动使用专栏选择的存档模型

## 文件结构

```
backend/uploads/archives/存档名/
├── 存档名 W              # 词嵌入模型文件
├── 存档名 N              # 诊断模型文件
├── knowledge_mapping.txt # 知识点映射文件
└── archive_info.json     # 存档信息
```

## API 接口

### 获取存档列表
```
GET /api/archives?teacher_id={teacher_id}
```

### 创建存档
```
POST /api/archives
Params: name, teacher_id
```

### 上传文件并训练
```
POST /api/archives/{archive_id}/upload
Files: knowledge_mapping, data_files[]
```

### 获取存档详情
```
GET /api/archives/{archive_id}
```

### 删除存档
```
DELETE /api/archives/{archive_id}
```

## 故障排除

### 训练失败
1. 查看存档详情的训练日志
2. 检查数据格式是否正确
3. 确保 NeuralCDM_plus-main 目录完整

### 模型无法使用
1. 确认存档状态为"已完成"
2. 检查模型文件是否存在
3. 查看训练日志是否有错误信息
